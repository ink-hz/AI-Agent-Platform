# Unified Agent Observability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend AI Agent Platform from a nine-MetaBot overview into a read-only eleven-Agent observability product with daily FAE/ADMIN synchronization and unified Sessions, Turns, Traces, Feedback, Reviews, and Flywheel views.

**Architecture:** Keep native MetaBot schemas unchanged, mirror production FAE and ADMIN data into isolated local PostgreSQL schemas once daily, and expose all sources through a canonical repository and API. Health remains independent and updates every 60 seconds; FAE spans come from its redacted `traces.jsonl`, while source-specific fields remain in `details` rather than being discarded.

**Tech Stack:** Python 3.11, FastAPI, psycopg 3, PostgreSQL, httpx, SSH, macOS Keychain/launchd, React 19, TypeScript, Vite, Vitest, pytest.

## Global Constraints

- The Platform is read-only; it does not modify prompts, knowledge, tools, Reviews, or releases.
- Do not create new public Session, Review, or ADMIN endpoints on `47.106.112.69`.
- Poll health every 60 seconds and synchronize business data once per day.
- Preserve questions, answers, descriptions, titles, and source content exactly as stored; do not translate them.
- Keep all nine existing MetaBot processes running throughout deployment.
- Production PostgreSQL access is read-only and credentials must never be printed or committed.
- Preserve the existing raw MetaBot Evidence privilege boundary.
- Preserve unrelated dirty worktree files in every repository.
- `Conversations` means answered Turns; display Sessions separately.
- A missing capability is `unavailable`, not an empty successful result.
