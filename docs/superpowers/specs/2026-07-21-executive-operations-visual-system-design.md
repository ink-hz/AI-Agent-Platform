# Executive Operations Visual System Design

- 日期：2026-07-21
- 状态：视觉方向已确认
- 方向：B — Executive Operations
- 范围：Agent Overview 首页的完整视觉层级

## 1. 目标

解决当前页面颜色过淡、字号过小、卡片过薄的问题，让 Agent Platform 呈现清晰、有重量、有业务价值感的 operations 产品形象。

本次不是简单放大所有元素，也不改成深色监控大屏。设计保持白色专业界面，通过更高文字对比、更明确的卡片边界、更有力度的阴影、稳定的字号阶梯和更充实的内部结构建立视觉层级。

## 2. 设计原则

1. 正文与解释文字首先保证可读性，不使用接近背景色的浅灰。
2. 卡片必须拥有可辨识的边界、表面和层级，不依赖 hover 才显现。
3. 数字、Agent 名称和状态是主要信息，视觉重量高于标签和解释。
4. 不用大面积渐变、玻璃拟态、霓虹光效或深色大屏制造虚假“高级感”。
5. Agent 业务色用于身份和关键指标，不覆盖整张卡片。
6. 手机端通过重排保证阅读，不通过继续缩小字体容纳内容。
7. 不改变页面信息架构、真实数据、Agent 状态和只读行为。

## 4. 页面层级

### 4.1 顶部导航与 Hero

- 顶栏高度从 `68px` 增至 `72px`。
- 品牌名称为 `16px`，导航为 `14px`，Read-only 标签为 `12px`。
- Hero 标题最大 `44px`，介绍为 `16px`，行高 `1.65`。
- 运行状态胶囊使用更深文字、更明确边框和 `0 8px 22px rgba(20, 45, 78, 0.09)` 阴影。
- Hero 与摘要之间保留足够空白，但不加入额外装饰区块。

### 4.2 汇总卡

- 最小高度 `150px`，内边距 `24px`。
- 边框使用 `--line`，圆角 `16px`。
- 常态阴影为 `0 12px 30px rgba(21, 51, 88, 0.09)`。
- 指标标签 `14px / 700`，指标数字 `42px / 780`，说明 `13px`。
- 每张卡保留右下角色块，但降低装饰面积，避免浅色圆形吞掉文字对比。
- Online 使用绿色，Total Conversations 和 Last 7 Days 使用品牌蓝；颜色只强调数字和小面积装饰。

### 4.3 趋势与排行卡

- 容器最小高度 `330px`，内边距 `26px`。
- 边框和阴影与汇总卡一致。
- 标题 `20px`，右侧汇总/说明 `13px`。
- 趋势柱使用更饱和的蓝色，轨道与网格线提高可辨识度。
- 柱值不再默认完全隐藏；桌面端使用 `12px` 常显，避免必须 hover 才能读数。
- 日期为 `12px`。
- 排行 Agent 名称为 `14px`，对话数为 `13px`；进度条高度从 `5px` 增至 `7px`。
- 排行 Avatar 从 `39px` 增至 `44px`。

### 4.4 Agent 卡片

- 桌面继续使用三列布局，保持一张卡代表一个 Agent。
- 内边距从 `21px` 增至 `24px`，圆角 `17px`。
- 边框为 `1px solid #c8d4e2`。
- 常态阴影为 `0 14px 34px rgba(20, 51, 89, 0.11)`；hover 阴影只做轻微增强。
- 顶部业务色条从 `4px` 增至 `6px`。
- Avatar 从 `45px` 增至 `52px`，文字至少 `13px`。
- Domain 为 `11.5px / 800`，Agent 名称为 `19px / 780`。
- 状态标签为 `12px`，扩大内边距和状态圆点。
- 中文介绍为 `14px`，颜色使用 `--ink-soft`，行高 `1.65`。
- 使用量区域上下内边距增至 `18px`，标签 `11.5px`；累计对话 `34px`，近七天 `19px`。
- Uptime、Last Activity 的标签为 `11.5px`，值为 `13px`。
- Recent 区域使用 `--surface-soft`，内边距 `13px`；标签 `11.5px`，正文 `13px`。
- Marketing Prospecting 等较长名称必须完整优先展示；卡片宽度不足时允许自然换行，不使用 `9px` 字体或强制截断解决。

## 5. Agent 业务色

保留九类业务色，但提高色条和关键指标的饱和度。浅色 Wash 只用于 Avatar 和状态背景。

- Collaboration：蓝
- People：紫
- Prospecting：橙
- Inbound：绿
- Voice：玫红
- Support：科技蓝
- Testing：石板灰
- Strategy：琥珀
- Intelligence：靛蓝

业务色文字必须使用当前深色 `--agent-text`，不得直接使用低对比浅色。

## 6. 响应式

- `> 1000px`：汇总四列、Agent 三列。
- `721–1000px`：汇总两列、Agent 两列、洞察单列。
- `<= 720px`：汇总单列、Agent 单列；导航折叠保持现状。
- 移动端正文、说明和标签仍遵守最低字号，不把桌面字号统一乘以缩放比例。
- Hero、区域标题和卡片内边距可以适度减少，但文字对比度和卡片边界不降低。

## 7. 状态与异常

- Active、Online、Degraded、Offline、Checking 继续同时使用文字、颜色和圆点，不以颜色作为唯一状态信息。
- 错误 Banner 使用实色浅背景和明确边框，字号不低于 `14px`。
- 加载和空状态的标题不低于 `20px`，说明不低于 `14px`。
- 不改变任何数据不可用、过期或保留最后成功数据的逻辑。

## 8. 实现边界

- 主要修改 `webui/src/styles.css`。
- 如长名称换行需要，可对 `FleetAgentCard.tsx` 做最小结构调整，但不修改数据或行为。
- 不修改 API、Flywheel、AgentCatalog、统计口径、轮询逻辑或文案体系。
- 不增加图标库、CSS 框架、Web Font 或第三方视觉依赖。
- 不修改或重启 MetaBot；构建完成后仅重启 Platform。
- 保留当前工作区无关的未提交修改与未跟踪文件。

### 8.1 子页面审查结论

当前代码中不存在真实子页面：`Overview / Agents / Sessions / Flywheel` 仅为非交互导航文字，前端没有 Router 或路径切换，后端也只有 `GET /api/fleet/overview`。因此本次不创建无数据、无交互的占位页面。

Executive Operations 同时作为未来子页面的强制视觉基线：

- 页面标题、正文、辅助说明和最小标签沿用本规格字号阶梯。
- Agent 详情、Session 列表、对话检查器和 Flywheel 面板沿用本规格色彩 Token、边框和表面层级。
- 列表行、详情面板和检查器正文不低于 `13px`，字段标签不低于 `11.5px`。
- 重要实体名称不依靠省略号隐藏；优先换行或调整布局。
- 空状态、错误状态和数据过期状态沿用本规格的高对比提示规则。
- 未来建设真实子页面时必须单独完成数据、交互和信息架构规格，不以静态外壳替代产品功能。

## 9. 验收标准

1. Agent 卡片名称、状态、介绍、使用量和最近信息无需放大浏览器即可快速阅读。
2. 生产页面不存在低于 `11.5px` 的可见文字规则。
3. 汇总卡、洞察卡和 Agent 卡在无 hover 状态下拥有清晰边界与层级。
4. 趋势值默认可见，排行名称和数值具有足够字号。
5. 三列、两列和单列布局均不截断重要名称或依靠极小字体容纳内容。
6. 页面保持白色 Executive Operations 风格，不变成深色大屏。
7. 所有现有功能、真实数据、错误降级和只读边界保持不变。
8. 前端测试、生产构建、依赖审计和线上验收通过。
