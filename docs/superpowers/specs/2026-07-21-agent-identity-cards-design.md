# MetaBot Agent 身份卡片视觉改造

## 背景

当前监控页准确展示了 9 个 MetaBot 实例的运行状态，但实例卡片以端口、运行时长、延迟和检测时间为视觉主体，更像服务器资源列表。迁移前的 Agent Portal 通过图标、名称、领域、简介和指标标签，使“一张卡就是一个 Agent”的关系非常清晰。

## 目标

在不恢复入口和控制功能的前提下，让每个监控目标重新具有独立的 Agent 身份感，同时补足看板用途说明。

## 页面结构

1. 顶部保留品牌栏、集群标题和整体健康灯。
2. 标题下增加紧凑的“看板说明”，明确监控范围、10 秒刷新、健康探测方式和只读边界。
3. 保留集群概览，但压缩摘要卡的高度，让 Agent 卡片成为页面主角。
4. 实例区使用身份化 Agent 卡片：
   - 左上为带主题色的文字图标；
   - 中间为友好中文名称、业务领域和一句职责介绍；
   - 右上为文字状态徽章；
   - 运行时长、响应延迟和 API 端口使用轻量指标标签；
   - 底部显示进程名、最后检测时间、数据新鲜度或探测错误。

## Agent 身份文案

身份信息仅由实例 ID 映射，不改变后端运行契约：

| 实例 ID | 展示名称 | 领域 | 图标 |
| --- | --- | --- | --- |
| `feishu-default` | 飞书默认助手 | 通用协作 | 飞 |
| `hr-bot` | HR 助手 | 人力资源 | HR |
| `marketing-prospecting-bot` | 营销拓客助手 | Marketing · Prospecting | 拓 |
| `marketing-inbound-bot` | 营销获客助手 | Marketing · Inbound | 入 |
| `marketing-voice-bot` | 营销语音助手 | Marketing · Voice | 声 |
| `fae-bot` | FAE 技术助手 | 技术支持 | FAE |
| `test-bot` | 测试助手 | 开发验证 | 测 |
| `marketing-gtm-bot` | GTM 策略助手 | Marketing · GTM | GTM |
| `marketing-intelligence-bot` | 市场情报助手 | Marketing · Intelligence | 情 |

未知实例使用运行契约中的名称，并生成通用“AI”图标和“MetaBot 实例”领域，保证契约动态新增实例时页面仍可用。

## 交互与边界

- 卡片不可点击，不包含链接、按钮或悬浮操作。
- 页面不调用任何控制接口，不执行重启或任务下发。
- 状态不能只依赖颜色，继续显示健康、异常、离线或检测中文字。
- 数据过期和接口失败行为保持不变。
- 移动端改为单列，身份、状态和指标不得溢出卡片。

## 验收标准

- 已知 9 个实例均展示友好名称、领域、简介和独立文字图标。
- 未知实例具有安全的通用回退身份。
- 页面明确写出“10 秒刷新”和“只读探测，不控制 Agent”。
- 实例卡片没有入口链接或控制按钮。
- 原有轮询、异常保留、状态判定和过期提示测试继续通过。
- 前端测试和生产构建通过，部署后 9 个 MetaBot 健康状态保持不变。
